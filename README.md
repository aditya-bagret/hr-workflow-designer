# HR Workflow Designer — Tredence Case Study

A production-grade visual HR Workflow Designer with a **FastAPI + SQLite backend**, AI assistant, live simulation, undo/redo, auto-layout, and workflow templates.

---

## Quick Start (Both servers in one command)

```bash
chmod +x start.sh
./start.sh
```

Or manually:

```bash
# Terminal 1 — Backend (FastAPI + SQLite)
cd backend
pip install -r requirements.txt
python3 main.py
# → http://localhost:8000

# Terminal 2 — Frontend (Vite + React)
npm install
npm run dev
# → http://localhost:5173
```

|--------|----------|-------------|

## What's Implemented

### Core Requirements ✅
- React 18 + Vite + TypeScript — zero-config, hot reload
- React Flow canvas with drag-and-drop from node palette
- 5 typed custom nodes: Start, Task, Approval, Automated Step, End
- Per-node configuration forms — dynamic, controlled components, full TypeScript types
- Mock API layer (GET /automations, POST /simulate) backed by real FastAPI server
- Workflow Sandbox with step-by-step execution log and live node animation

### Bonus Features ✅ (All listed bonuses implemented)
- **Export/Import workflow as JSON** — toolbar buttons + Ctrl+E
- **Undo/Redo** — Ctrl+Z / Ctrl+Y, 60-step history in Zustand. Fixed: uses `onNodeDragStop` not `onNodesChange` so dragging nodes is correctly tracked
- **Minimap + zoom controls** — bottom-right of canvas
- **Validation errors visually on nodes** — error/warning badge on each node card, full message shown in form panel when selected
- **Auto-layout** — Dagre.js tree positioning, Ctrl+L
- **Workflow Templates** — 3 pre-built HR workflows (Onboarding, Leave Approval, Document Verification)

### Differentiator Features 🚀
- **🤖 AI Workflow Assistant** — Claude-powered chat panel. Sends full graph context (nodes, edges, types, assignees, params) on every message. Quick prompt buttons for common queries
- **💾 Backend Persistence** — FastAPI + SQLite. Save, load, update, delete named workflows. Online/offline indicator with graceful degradation when backend is not running
- **⚡ Live Canvas Simulation** — nodes light up with glow + shimmer one-by-one on canvas as simulation executes. 500ms per step
- **🧭 Working Sidebar Navigation** — all 10 nav items render real content: Dashboard (activity feed, stats), Analytics (live node distribution chart), Compliance/Scheduler/etc. (scoped placeholder modules), Workflows (full backend CRUD UI)

## Key Design Decisions

**Undo/Redo via `onNodeDragStop`** — React Flow's `onNodesChange` fires on every position tick during drag (100+ events). Calling `pushHistory` there would blow out the history stack. Instead, `onNodeDragStop` fires exactly once when the user releases the mouse, creating a single clean snapshot. All other mutations (add, delete, connect) also push history.

**Discriminated union types** — `WorkflowNodeData` is `StartNodeData | TaskNodeData | ...`, each interface `extends Record<string, unknown>` for React Flow's generic constraint. The form panel and validation hook switch exhaustively on `data.type` — adding a new node type is a 3-file change: type interface → form component → switch case.

**Zustand history stack** — undo/redo state lives inside the same Zustand store as the canvas. `structuredClone` ensures snapshots are deep copies. Capped at 60 entries. `canUndo()` and `canRedo()` are getter functions (not derived state) to avoid re-render loops.

**Backend + graceful degradation** — `backendApi.ts` wraps every call in typed `fetch`. `SavedWorkflowsPanel` calls `checkHealth()` on mount and shows a cloud/offline icon. If the backend is down, all other canvas features keep working — the mock API is still the simulation fallback.

**Live simulation animation** — `simulatingNodeIds: string[]` in store. The sandbox iterates steps with async delays, updating the array one ID at a time. `BaseNode` reads this directly from the store to apply a glowing border + CSS shimmer animation — no prop drilling.
